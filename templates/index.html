Minha primeira API CRUD

Essa é uma API para testar e compreender como criar uma API com principio de um CRUD usando como base de dados um arquivo JSON
Os endpoints:

/todos

Esse endpoint é do metodo GET e retorna todos os dados cadastrados na base de dados.

/adicionar-usuario

Esse endpoint é do metodo POST e adiciona um usuarioa base de dados dave enviar como body na requisição e deve estar em JSON:

{
"nome": " ",
"email": " ",
"senha": " "
}


Os estatus de retorno possivel são 200 OK o 400 BAD REQUEST

/consulta-especifica

Esse endpoint é do metodo GET e precisa de argumentos na URL para a requisição os argumentos são:
typeSearch: id_user , nome , email. Esses são as opções de busca.
data: aqui é a informação que sera usada para filtrar.
Exemplo: http://127.0.0.1:5000/consulta-especifica?typeSearch=id_user&data=1
Os estatus de retorno possivel são 200 OK o 400 BAD REQUEST.

/alterar-user

Esse endpoint é do metodo POST e precisa de argumentos na URL para a requisição os argumentos são:
typeSearch: id_user , nome , email. Esses são as opções de busca.
data: aqui é a informação que sera usada para filtrar.
Exemplo: http://127.0.0.1:5000/alterar-user?typeSearch=id_user&data=1
E também precisa de um body com um JSON contendo os novos dodos:

{
"nome": " ",
"email": " ",
"senha": " "
}


Os estatus de retorno possivel são 200 OK o 400 BAD REQUEST

/apaga-user

Esse endpoint é do metodo DELETE e precisa de argumentos na URL para a requisição os argumentos são:
typeSearch: id_user , nome , email. Esses são as opções de busca.
data: aqui é a informação que sera usada para filtrar.
Exemplo: http://127.0.0.1:5000/apaga-user?typeSearch=id_user&data=1
Os estatus de retorno possivel são 200 OK o 400 BAD REQUEST
